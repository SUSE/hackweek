    • ssh scan blocker

    Tags: security, Network, Server, SSH, openSUSE, SLES, SLED, InProgress

    Description

    I noticed on my server several break-in attempts by ssh. The bad guys try out random accounts (in reality there seem to be lists, which got repeated and extended month by month) with random passwords.

    ssh seem not to provide anything, which blocks any such break-in attempts after a specific time or number of failed login attempts.

    To avoid DoS: A mechanism to re-allow blocked hosts after some time-out is needed. A white-list of hosts is needed.

    To distinguish from iptables functionality: it's not restricted to a specific protocol (ssh) and the number of bad logins should be weighted (failed root accounts should activate blocker earlier).
