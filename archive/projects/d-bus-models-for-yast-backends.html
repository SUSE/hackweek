    • D-Bus models for YaST backends

    Tags: Idea, YaST

    Description

    It would be nice to have a D-Bus API to access the YaST backends as model for views in the model-view sense. By encapsulating the access to configuration files, the package manager, system services etc. in a D-Bus API there arise some interesting options, e.g. access to YaST functionality independent of programming languages or special frameworks, concurrent access to the same data by different views, clear separation between data processing and user interface. By providing the typical model notifications of data changes it would be possible to dynamically update views and create a rich experience for the user. As D-Bus is widely supported by current desktop frameworks it would be easily possible to write panel applets and other small tools interacting with or showing system data or even integrate it with applications, so that for example a web page editor could remote control a web server used to preview the pages.

    People

    Cornelius Schumacher originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
