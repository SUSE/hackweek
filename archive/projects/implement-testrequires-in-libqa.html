    • implement test:requires in libqa

    Tags: Idea, Community, Security, QA, Kernel, Desktop, Server, DevTool, openSUSE, SLES, SLED, SLERT, SLEPOS, SLETC, HelpWanted

    Description

    QA uses ctcs/stasis as their primary test execution frameworks. Both shims provide a simplistic job execution language/description format. Tests that are ment to be run in the framework are supposed to make use of "libqa", a unified API for common QA tasks.

    So, at a time when we have our trillions of automated test cases running, wouldn't it be nice to only run tests where the prerequisites are fulfilled? Run storage tests only when sufficient disk space is available, run bonding tests only when a corresponding device is configured.

    We are currently missing the functionality to check for certain prerequisites through a consistent API.

    People

    Olli Ries originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
