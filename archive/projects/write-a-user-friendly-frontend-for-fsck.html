    • Write a user friendly frontend for fsck

    Tags: TryMe, filesystem, usability

    Description

    When a fsck fails during boot most non technical users right now are in deep ***. The system asks lots of obscure question which the user usually doesn't know anything about. This can still happen with journaling file systems e.g. after an IO error.

    My suggestion would be to write a higher level frontend for e2fsck that gets activated during a forced fsck.

    The frontend could be console mode, otherwise an X server would need to be started (would need to interact nicely with the splash screen then). Or perhaps starting an X server isn't too bad.

    It should first give a clear high level message about what happened (e.g. Checking your file system because of I/O error) When there is any error that normally requires user to answer a question it should suggest to either always answer y or allow the user to answer all the question if they're an expert.

    Then afterwards it should give a summary of the files in /lost+found (size/ file(1) output etc.)

    There could be probably be more done to improve this.
