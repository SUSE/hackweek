    • Clone installed system for testing

    Tags: Idea, QA, Virtualization, openSUSE, Idea

    Description

    The idea is to create a tool that will automagically create a virtual machine from installed system. The VM then can be used for real-life update testing of beta distributions.

    The VM shold run on the original machine, preferably without impact to other services and without reboot. The only requirements should be root account and enough disk space.

    It should work this way:

    1. boot an image on target machine (virtual or physical)

    2. target asks for root ssh password for source machine

    3. target analyzes the source machine over ssh and propose the migration

    4. copy whole partitions or all files over ssh

    5. necessary adjustments on target

    This setup should run everywhere (xen, virtualbox, vmware, physical test machine) and with minimal influence on source machine. It makes the deployment in the openSUSE community easy.

    People

    Vladimir Nadvornik originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
