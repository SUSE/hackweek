    • Improve YaST computer languages support

    Tags: Management, SLES, InProgress, InBuildService, yast

    Description

    YaST has to evolve. There are lot of ideas around system management:

    The Web YaST:

    http://idea.opensuse.org/content/ideas/web-administration-tools-reusing-yast-modules

    A nice idea, but requires YaST as a daemon.

    Another idea: http://idea.opensuse.org/content/ideas/yast-as-a-webservice

    YaST has a lot of backend code. If we could take advantage of this code from modern languages we could imagine a smooth transition. This would also help the ongoing effort on the YaST UI:

    http://idea.opensuse.org/content/ideas/making-the-yast2-ui-usable-without-all-the-ycp-stuff

    The idea needs research on:

    * Make YaST API accesible from any language

    * Make some language be able to easily call YCP/Perl modules

    * Make YCP code be able to call other languages than perl.

    ----

    The idea is implemented! http://duncan.mac-vicar.com/blog/archives/245

    Temporary svn: http://svn.opensuse.org/svn/yast/branches/tmp/dmacvicar/ruby-yast/

    Now in build service: Package yast2-ruby-bindings in http://software.opensuse.org/download/home:/dmacvicar/
