    • Make GTK# work with multiple application domains

    Tags: Idea, Mono

    Description

    GTK# can only run in the main app domain of a process. The idea is to make the necessary changes so that GTK# code can run in other app domains. For example, a widget could be created in a secondary domain and added to a window created in the main domain. This would allow dynamic loading/unloading and app-domain isolation of assemblies implementing GTK# widgets, and would be very useful for GUI designers or other kind of application which require app-domain isolation for add-ins.

    People

    Lluis Sanchez originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
