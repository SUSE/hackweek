    • universal systemwide help agent framework

    Tags: Idea, Desktop, UnixFeature

    Description

    A systemwide operating universal and easy extensible help agent framework (with three layers) to give users a helpsystem not specific to a special app or desktop.

    Frontends could be textbased (for usage on pure textconsole) or even as interactive avatar. Backends to scan and analyze the system, for integrating additional system-"knowledge" could build a easily extensible framework to find "most common" errors in system.

    A daemon running in background - looking for various suspicious or even dangerous situations - could inform the user of possible problems the system has. The other way round the user could query the framework - via the various frontends - if e.g. the harddisk is ok as it is sooo slow .. which will make the framework to look for backends to scan and analyze the disk, telling the user "don't know yet, but i can check for it via "foobar-test" and "foobar-benchmark" if it is working as appropriate.. do you want me to proceed (y/n)?" .. and so forth

    The Frontends need to do the parsing themselves, trying to identify what the user asks for, comunicating with the framework with a defined API .. _but_ the framework could also help the frontends via special knowledge-backends that e.g. could hold common phrases for often arising problems.

    The framework could eventually also include some sort of automated learning .. associating found system states with arising problematic symptoms.
