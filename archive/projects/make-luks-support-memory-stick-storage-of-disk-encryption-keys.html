    • Make LUKS Support Memory Stick Storage of Disk Encryption Keys

    Tags: Idea, Security, Desktop, SLED

    Description

    SLED uses LUKS to provide volume/disk encryption. The main threat this defends against is stolen laptops; it makes it hard for the person who stole your machine to get your data.

    The problem is that LUKS by default stores the disk encryption key on the disk itself, and then depends on the quality of your pass phrase to defend your data. A much more secure approach would be to store the volume encryption key on a memory stick or a server.

    To make it work, LUKS and/or YaST for LUKS needs support for the key not being on the volume. E.g. when you wake the laptop up, it should demand the key, and if it isn't there, sit and wait for you to insert your memory USB stick.

    People

    Crispin Cowan originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
