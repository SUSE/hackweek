    • Job Scheduler

    Tags: Idea, Server, Management

    Description

    Cron is OK for scheduling simple jobs, not caring about dependencies at all. In enterprise IT environments, it's common to find job schedulers that can do things like: "Launch this process whenever these two other processes have ended OK and it's not later than 02.00 AM, and if it finishes Not OK, send a warning e-mail to sysadmins"

    Next level should be make it work in a distributed environment.

    There have been some implementations of this idea, but as far I can recall no one is alive as a maintained opensource project right now.

    People

    Serxio Barral originated this idea. No one is currently working on implementing this idea (it's up for grabs!).
