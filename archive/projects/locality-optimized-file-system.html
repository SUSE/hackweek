    • Locality optimized file system

    Tags: Idea, Performance, filesystem

    Description

    Applications spend much of their time doing slow disk I/O, often reading many small files off the disk. A file system that could monitor file usage to identify "related" files (files often read sequentially by programs) and cluster them together, contiguous on the disk, could improve performance of many applications quite a bit.

    People

    Lubos Lunak has talked about this idea before. Michael Meeks has worked on I/O profiling tools that could be very useful here. Federico Mena-Quintero has made similar performance optimizations.
