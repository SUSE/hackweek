    • Implement Desklets using Moonlight

    Tags: Idea, InProgress, Mono, Desktop, Web20, DevTool, Graphics, javascript, TryMe, HelpWanted, Judgeme, JudgeBest, JudgeDesktop

    Description

    OSX and Vista implements some nice small desktop applications (aka applets/docklets/gadgets/desklets). As I have been working in the past few weeks on Moonlight (Free implementation of Silverlight), and seeing as Moonlight is a perfect way to get things like OSX Dashboard or gDesklets.

    Why not use gDesklets? Because Moonlight give us a way to write much more rich Desklets using animations and storylines and full access to C#

    Moonlight (although Gtk# might be used in a few places) will be used to get effects and for rapid application development.

    Meet us in channel #desklets in irc.gnome.org

    You can find the sample desklets directory in moon/desklet

    Basic goals:

    - Applications will be basic XAML shells that can use custom widgets. - See moon/desklet/lameclock for a sample desklet using XAML and Moonlight. - See moon/desklet/lameclock for some sample animation for configuration. - To launch applications, we will use mopen, part of Moonlight. - you can use mopen with the -transparent flag to get alpha blended widgets on the screen (under implementation now). - Read the mopen.1 man page for details about code sharing, VM sharing, and how to launch new apps. - "Gadgets" or "Widgets" will basically be simple XAML apps, see Surface for a sample of these can be done. - Since we will be using "mopen" infrastructure, the entire desklet has to be stored in a directory, with an entry file called "default.xaml".

    Some other features: - Write a gadget launcher, this probably needs to be done once we have some gadgets, handle install ationand add desklets to desktop, this application will use mopen to share one Mono VM for all desklets. Multi domains will also needed. - The entire desklet can be written with Moonlight/XAML and is preferred over Gtk# because its content can be easily animated using XAML's animation features. - Create some sample desklets: - Multi clock, an digital clock with multiple themes. - Calendar desklet, maybe just a transluced calendar with some options like amount of months (1,3,6, year) - Slide show desklet, an slide show desklet application to show images from dir, time to change images can be configured, also we can show images from picasa or flickr albums. - Some other desklet ideas will accepted and implemented based on number of hackers that join with us.

    Things to decide:

    - Best way to package desklets (zip, gz or bz). Zip seems to be the best option, since mono ships with SharpZip lib and that way we have no external dependencies. - ...

    These are the Desklets we either plan to implement or are already done:

    - Calendar - Done, in opaque and glassy versions. Calendars integreate with Google Calendar highlighting days with relevant event. - Digital Clock - Done - Analog Clock - Done, in opaque and glassy versions. - CPU monitor - Done. - Network traffic monitor - Done, very simple monitoring of the current upload/download Kbps. - Calculator - Done, features only elementary math. - Weather - Done, queries online data. - Slideshow - Done, integrated with Picasa, it slides thru a photo set. - Rule - Done, allows one to measure the whole desktop! - Video Player - Done, features only simple stop/pause/start controls.

    The team is using the irc channel #desklets at irc.gnome.org to meet and discuss the subject, everyone is invited to join us and chat about all our nice Desklets :)
