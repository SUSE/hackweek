    • Dependency graphing in package database

    Tags: InProgress, Management, HelpWanted, devtool

    Description

    Getting package dependencies right is hard sometimes. Many packages drag in megabytes of dependant packages - without real need in a lot of cases. Update dependencies are a different kind of beast and even harder in case of package renames, splits, merges, etc.

    The goal is to add a 'dependency graph' button to the package database (or to autobuild) to help packagers visualizing package dependencies (and summarized install sizes).

    People

    Stefan Haas and Stefan Schubert already work on this. Getting autobuild and pdb integration right and making the tool useful for packagers still need some help.
