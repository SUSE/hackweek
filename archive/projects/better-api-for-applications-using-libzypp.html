    • Better API for applications using libzypp

    Tags: Idea, Libzypp, YaST, HelpWanted, HW-Hacker

    Description

    Currently libzypp does not offer (or offers only very limited scope of) high-level functions for common package (patch, pattern) operations such as 'list all patterns','list packages that belong to a pattern' 'list all objects fulfilling certain criteria' etc.

    If an application (such as UI library, zypper etc.) wants to use libzypp, it needs to implement these functions by itself. As a result, the same functionality is (in better case) implemented by the code that is copy-pasted n times between n applications, or (in worse case) implemented inconsistently in n different ways.

    The idea is to design API that would 1) make libzypp functionality easily accesible to the applications by offering simple functions returning data types convenient for the application 2) implement these functions once only (in libzypp or in a shared library over libzypp) so that applications can simply call them and do not have to care about libzypp implementation details

    People

    Katarina Machalkova originated this idea.

    Authors of applications using libzypp (HuHa - yast2-qt, Michael Meeks, Ricardo Cruz -yast2-gtk, Jano Kupec - zypper) might be interested as well

    Katarina Machalkova would like to start with going through the code of y2-ncurses and y2-qt, to identify code consisting of libzypp functions only and evaluate which of this could be possibly moved to libzypp (a wrapper library over libzypp respectively).

    The next step will be to suggest suitable prototypes for these 'to-be-moved' functions so that they are conveniently handled by UI library.

    Help of libzypp hackers wanted!

    Jano Kupec is planning to implement part of this idea (see comments).
