    • Develop PHP library for Testopia API

    Tags: Done, QA, Testopia, PHP, Prize, JudgeMe, JudgeQA

    Description

    Testopia offers an access via XML-RPC API. Having a PHP library to access this API would offer websites (and commandline tools) read/write access to Testopia. The idea is to develop such a library.

    Working with such a library would save QA team a lot of work and a lot of time.

    It's - internal - available at http://w3.suse.de/~holgi/phpTestopialib.tgz

    Information in telegraphic style

    Testopia is an enhancement of Bugzilla. We use it to create TestPlans and track testing. It can be controlled by a webinterface -> which works fine for people but bad for machines. So Testopia got a XML-RPC interface.

    My project was to develop a library which 'talks' to that Testopia interface. I decided to use PHP because it offers access from our webbased tools as well as from the commandline.

    The library contains 72 functions and offers nearly all Testopia operations which are available via the webinterface. It is only limited by the Testopia API itself.

    This library offers a bunch of new possibilities for our QA teams: * creation and modifing of TestPlans and TestRuns * creation and modifing of TestCases * creation and modifing of Test Results And all this by our scripts or tools.

    To understand the benefit following example will help you.

    Using the library with our QA tool HAMSTA:

    With HAMSTA we can start testsuites on several testmachines via a webinterface. During the testsuites are running they execute a bunch of Testcases and store the results in result file. This files will be colleted by the HAMSTA tool afterwards but the results have to be entered into Testopia by a person.

    With my library HAMSTA can be enhanced to: * create a new TestPlan and assign relevant TestCases * create a TestRun for every hardware architecture * collect the results of all tests and enter them into Testopia Creation of TestPlans and TestRuns will done automatically as well as entering all the results. So we save a lot of time in future and can – hopefully – take the time to develop more useful QA tools and libraries ...

    And best thing of project: it's finished and working fine!! :-)

    Here is a small demo: You can [[http://tidb.suse.de/testopia/demo.php|modify]] the status of a single TestCase of a certain TestRun from a non Testopia website and check result within [[https://bugzilla.novell.com/tr_show_run.cgi?current_tab=case_run&field-1-0-0=run_id&run_id=299&type-1-0-0=anyexact&value-1-0-0=299&pagesize=1&page=1|Testopia]]: * [[https://bugzilla.novell.com/tr_show_run.cgi?current_tab=case_run&field-1-0-0=run_id&run_id=299&type-1-0-0=anyexact&value-1-0-0=299&pagesize=1&page=1|Testopia view]] * [[http://tidb.suse.de/testopia/demo.php|Demo page]] (just available internal) Of course this is just a simple demo. More complex operations are possible.
