    • Torrent-Style Multimedia player plugin for browsers

    Tags: Idea, Performance, Web20, Mozilla, network, multimedia, filesharing

    Description

    Consider the scenario:

    Nat gives a demo of SLED 11 and its cool features. This is recorded and the video is uploaded in a web-server. Now Novell employees can either

    1) Download the video and watch it offline (or)

    2) Stream through their browser (.FLV etc)

    As per our plan, the server will split the video file into manageable chunks and sends the chunks to the clients requested.

    It is highly improbable that all employees will start watching the video at the same instant. Subsequent new user requests to view the file can be redirected to any of the old clients who have already downloaded the chunks. The server should track which users have which chunks and should be able to redirect any further user requests.

    The server should have a decent distribution algorithm so that not a single/group of clients are bombarded with many requests by many clients.

    Optionally, the server may choose to send the 1st chunk alone by itself.

    This idea is very much like bit-torrent and is essentially trying to adapt the bit-torrent for multimedia streaming.

    This should hopefully reduce the server traffic.
